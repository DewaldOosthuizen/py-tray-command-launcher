        # Raise PBKDF2 Iteration Count to Meet Current OWASP Minimums in FileEncryptor

## Overview

`EncryptionWorker._derive_key` in `src/modules/file_encryptor.py` uses PBKDF2-HMAC-SHA256
with a hard-coded iteration count of 100,000. The OWASP Password Storage Cheat Sheet
(updated 2023) raises the minimum for PBKDF2-HMAC-SHA256 to 600,000 iterations to resist
offline brute-force attacks on commodity hardware. Because the current value is embedded
directly in the call site with no named constant and is not persisted alongside the salt,
it cannot be evolved without breaking existing encrypted files silently. This change raises
the iteration count to the OWASP minimum, persists it in the `.salt` file so decryption
is always self-describing, and provides a legacy fallback for files encrypted before this
change.

## Issue 1 — Hard-coded iteration count below OWASP 2023 minimum

**File:** `src/modules/file_encryptor.py` (line 53)

**Problem:** `iterations=100000` is a magic number sitting at roughly 1/6 of the OWASP
2023 minimum. An attacker with a modern GPU can perform millions of candidate-password
trials per second at this work factor, materially reducing the protection offered to users
who store SSH keys, credential files, or private documents.

**Fix:**

Before:

```python
# line 29-30 (module-level constants)
SALT_FILE_SUFFIX = ".salt"
ENC_FILE_SUFFIX = ".enc"

# line 46-56
def _derive_key(self, password: str, salt: bytes) -> bytes:
    """Derive encryption key from password using PBKDF2."""
    password_bytes = password.encode('utf-8')
    kdf = PBKDF2HMAC(
        algorithm=hashes.SHA256(),
        length=32,
        salt=salt,
        iterations=100000,
    )
    key = base64.urlsafe_b64encode(kdf.derive(password_bytes))
    return key
```

After:

```python
# module-level constants (after existing SALT_FILE_SUFFIX / ENC_FILE_SUFFIX)
SALT_FILE_SUFFIX = ".salt"
ENC_FILE_SUFFIX = ".enc"
# OWASP Password Storage Cheat Sheet (2023): minimum 600 000 for PBKDF2-HMAC-SHA256.
_PBKDF2_ITERATIONS = 600_000

# _derive_key gains an `iterations` parameter so the value stored in the .salt
# file can be round-tripped back into the KDF on decryption.
def _derive_key(self, password: str, salt: bytes,
                iterations: int = _PBKDF2_ITERATIONS) -> bytes:
    """Derive encryption key from password using PBKDF2-HMAC-SHA256.

    Args:
        password:   plaintext password supplied by the user.
        salt:       16-byte random salt generated at encryption time.
        iterations: PBKDF2 iteration count; defaults to _PBKDF2_ITERATIONS.
                    Pass the value read from the .salt file on decryption to
                    maintain backward compatibility with older encrypted files.
    """
    kdf = PBKDF2HMAC(
        algorithm=hashes.SHA256(),
        length=32,
        salt=salt,
        iterations=iterations,
    )
    return base64.urlsafe_b64encode(kdf.derive(password.encode("utf-8")))
```

## Issue 2 — Salt file does not persist iteration count, breaking forward compatibility

**File:** `src/modules/file_encryptor.py` (lines 163-171 encrypt path, lines 185-188 decrypt path)

**Problem:** The `.salt` file currently stores only the raw 16-byte salt. When the
iteration count changes (now or in any future upgrade), files encrypted with the old count
cannot be decrypted because the KDF will silently derive the wrong key. There is no
information in the file to detect or correct this.

**Fix:** Prepend a 4-byte big-endian uint32 containing the iteration count to the 16-byte
salt, producing a 20-byte `.salt` file. On decryption, branch on file length:

- 20 bytes: new format — read uint32 iterations, read 16-byte salt, call `_derive_key`
  with the stored count.
- 16 bytes: legacy format — assume 100,000 iterations (the old hard-coded value), call
  `_derive_key` with `iterations=100_000`.
- Any other length: corrupt or unrecognised format — emit an error and abort.

Before (encrypt, lines 163-171):

```python
if self.operation == 'encrypt':
    if self.is_folder:
        salt_file = os.path.join(self.file_path, '.encryption_salt')
    else:
        salt_file = self.file_path + SALT_FILE_SUFFIX
    with open(salt_file, 'wb') as f:
        f.write(salt)            # 16 bytes only
```

After (encrypt):

```python
import struct  # add to module-level imports

if self.operation == 'encrypt':
    if self.is_folder:
        salt_file = os.path.join(self.file_path, '.encryption_salt')
    else:
        salt_file = self.file_path + SALT_FILE_SUFFIX
    with open(salt_file, 'wb') as f:
        # 4-byte big-endian uint32 iteration count + 16-byte salt = 20 bytes total
        f.write(struct.pack(">I", _PBKDF2_ITERATIONS))
        f.write(salt)
```

Before (decrypt, lines 185-188):

```python
if os.path.exists(salt_file):
    with open(salt_file, 'rb') as f:
        salt = f.read()         # reads raw 16 bytes, no iteration metadata
    key = self._derive_key(self.password, salt)
```

After (decrypt):

```python
_LEGACY_ITERATIONS = 100_000   # value used by all files encrypted before this change

if os.path.exists(salt_file):
    with open(salt_file, 'rb') as f:
        raw = f.read()
    if len(raw) == 20:
        # New format: 4-byte big-endian uint32 + 16-byte salt
        stored_iterations = struct.unpack(">I", raw[:4])[0]
        salt = raw[4:]
    elif len(raw) == 16:
        # Legacy format: 16-byte salt only — iteration count was 100 000
        stored_iterations = _LEGACY_ITERATIONS
        salt = raw
    else:
        self.finished_signal.emit(
            False,
            "Salt file is corrupt or unrecognised format. Cannot decrypt."
        )
        return
    key = self._derive_key(self.password, salt, iterations=stored_iterations)
```

## Migration Note

Files encrypted with the previous release (iteration count 100,000, 16-byte `.salt` file)
are handled transparently by the legacy-fallback branch. No user action is required for
decryption. To benefit from the stronger work factor, users should decrypt existing files
with the old version (or this version's fallback path) and re-encrypt with the new version.
The README should document this path clearly as a breaking change for anyone copying `.salt`
files between machines or versions.
