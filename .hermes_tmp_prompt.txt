        # Fix QProcess Double-Start Bug in execute_command_process_silently

## Overview

`CommandExecutor.execute_command_process_silently` (line 39-43 of
`src/modules/command_executor.py`) calls `self.execute_command_process(app, command)`, which
already invokes `process.start()` on line 35. The method then unconditionally calls
`process.start()` a second time on line 42. Starting an already-running `QProcess` is undefined
behaviour in Qt: depending on the platform and Qt version it may spawn a second child process,
silently no-op, or leak a file descriptor. Every command routed through the silent path currently
runs twice, inflating the process badge count and producing unintended side-effects for
non-idempotent commands (file writes, network requests, cron-like triggers).

An existing unit test at line 92-100 of `tests/test_command_executor.py` asserts
`mock_process.start.assert_called_once()`, but because the mock intercepts both calls to the
same object the assertion passes while the production code remains broken.

## Issues

### Issue 1 — Redundant process.start() in execute_command_process_silently

**File:** `src/modules/command_executor.py` (lines 39-43)

**Problem:** `execute_command_process` starts the process on line 35 and returns the handle.
`execute_command_process_silently` calls `process.start()` again on line 42 without checking
whether the process is already running, causing the underlying shell command to be invoked twice.

**Fix:**

```python
# Before (lines 39-43)
def execute_command_process_silently(self, app, command):
    """Execute a command without showing the output."""
    process = self.execute_command_process(app, command)
    process.start()                                          # BUG: already started
    logger.debug("QProcess started silently for command: %s", command)

# After
def execute_command_process_silently(self, app, command):
    """Execute a command without showing the output.

    Delegates to execute_command_process, which starts the QProcess
    internally.  Do NOT call process.start() here again.
    """
    process = self.execute_command_process(app, command)
    # process.start() is already called inside execute_command_process.
    logger.debug("QProcess started silently for command: %s", command)
```

### Issue 2 — Unit test does not isolate execute_command_process from execute_command_process_silently

**File:** `tests/test_command_executor.py` (lines 92-100)

**Problem:** The test patches `QProcess` at the class level and counts `start` calls on the
mock. Because both the inner and outer `process.start()` calls hit the same mock object, the
`assert_called_once()` assertion passes even when the production code calls `start()` twice.
The test gives a false green and does not catch the double-start regression.

**Fix:**

```python
# Before (lines 92-100) — patches QProcess but does not isolate the inner call
def test_execute_command_process_silently_starts_process(self):
    """execute_command_process_silently should call start() on the QProcess."""
    mock_process = MagicMock()
    mock_app = MagicMock()

    with patch('modules.command_executor.QProcess', return_value=mock_process):
        self.executor.execute_command_process_silently(mock_app, "df -h")

    mock_process.start.assert_called_once()

# After — spy on execute_command_process to count start() calls precisely
def test_execute_command_process_silently_starts_process_exactly_once(self):
    """QProcess.start() must be called exactly once via the silent path."""
    mock_process = MagicMock()
    mock_app = MagicMock()

    with patch('modules.command_executor.QProcess', return_value=mock_process):
        # Spy on the inner helper to confirm it is called (and only it starts the process)
        with patch.object(
            self.executor,
            'execute_command_process',
            wraps=self.executor.execute_command_process,
        ):
            self.executor.execute_command_process_silently(mock_app, "df -h")

    # The process must have been started exactly once regardless of call path.
    self.assertEqual(mock_process.start.call_count, 1,
                     "QProcess.start() must be called exactly once on the silent path")
```
